4. ADD INCIDENT / TICKET CORRELATION LOGIC
-----------------------------------

When new controller events or telemetry changes arrive:
- try to find an existing open ticket for the same customer/site/service/device/link
- if found, attach IncidentCorrelation and avoid duplicate ticket creation
- if not found, create a new ticket or suggested incident ticket

Incident rules:
- if a device goes offline, likely create ticket
- if a primary transport goes down but backup remains active, create degraded incident and set failoverActive = true
- if both primary and backup are down, create outage incident
- if firewall HA failover event occurs, create high severity incident but not necessarily outage
- if event is informational only, do not auto-create critical incident

Use a reasonable rule-based engine for MVP.
