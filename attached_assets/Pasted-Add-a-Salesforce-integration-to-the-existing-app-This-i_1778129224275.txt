POST to {SALESFORCE_INSTANCE_URL}/services/oauth2/token

Store the returned access_token and instance_url in memory
Add token refresh logic if token expires
Add a testConnection() function that verifies auth works
Handle errors cleanly — if credentials are missing or wrong, fail gracefully with a clear error message

artifacts/api-server/src/connectors/salesforce.ts
Implement these functions:
testConnection()
syncAccounts()
syncContacts()
fullSync()
syncAccounts() should:

Query Salesforce Accounts via REST API
Endpoint: GET /services/data/v58.0/query?q=SELECT+Id,Name,BillingStreet,BillingCity,BillingState,BillingPostalCode,BillingCountry,Phone,Website+FROM+Account
Map each Account to your internal Customer model
Store externalSystem = 'salesforce'
Store externalId = Salesforce Account Id
Upsert by externalId — do not create duplicates
Store lastSyncedAt and syncStatus

syncContacts() should:

Query Salesforce Contacts via REST API
Endpoint: GET /services/data/v58.0/query?q=SELECT+Id,FirstName,LastName,Email,Phone,Title,AccountId+FROM+Contact
Map each Contact to your internal customer_contacts model
Link to the correct internal customer via the mapped AccountId
Store externalSystem = 'salesforce' and externalId
Upsert by externalId
